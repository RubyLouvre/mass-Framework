$.fn.keypress([ fn ])

描述：

为 keypress 事件绑定一个处理函数，或者触发元素上的 keypress 事件。

这个函数的第一种用法是 .bind('keypress', handler) 的快捷方式，第二种用法是 .fire('keypress') 的快捷方式。

当浏览器捕获一个元素键盘输入时keypress就会发送个元素。这是类似keydown事件，除了在关键重复情况。如果用户按下并按住这个键，一keydown事件触发一次， 但不同的是keypress为每个插入的字符都触发事件。此外，组合键（如Shift）会触发keydown事件，但不会触发keypress事件。

keypress事件处理程序可以连接到任何元素，但该事件只是发送到具有焦点的元素上。 在不同浏览器之间聚焦元素可以有很多，但是表单元素总是能获取焦点，所以对于此事件类型表单元素是最合适的。

如果需要任何按键被捕获（例如，要实现网页全局上的快捷键），附加此事件到document对象是非常有用的。 由于事件冒泡，所有的按键将使他们DOM方式注册到document对象上，除非明确停止。

要确定哪个键被按下，我们可以检测回调函数第一个参数event对象的which属性。此属性对应一个键盘上的按键，包括像箭头这种特殊键的代码， 为捕获实际输入文本， .keypress()可能是一个更好的选择。

值得注意的是，由于onkeypress不能对系统功能键进行捕获，导致event对象的which属性和onkeydown，onkeyup键盘事件中获取的which属性不同， 主要表现在onkeypress事件的which对字母的大小写敏感， 而onkeydown、onkeyup事件不敏感；onkeypress事件的which无法区分主键盘上的数字键和副键盘数字键的， 而onkeydown、onkeyup的keyCode对主副键盘的数字键敏感。 因此，当捕捉特殊按键，如箭头，.keydown()或.keyup()是较好的选择。

参数：

fn
可选。Funcion。回调函数。
例子

当一个按键在文本框内被按下的时候显示事件对象的所有属性.

$.require("ready,event", function() {
    $(".sample input").keypress(function(e) {
        var html = ""
        for (var i in e) {
            html += i + "   :  " + e[i] + "<br/>"
        }
        $(".sample").height($(".sample div").html(html).height() + 40)
    });
});
